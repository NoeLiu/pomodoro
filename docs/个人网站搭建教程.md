# 个人网站低成本搭建教程

> 基于 GitHub Pages + Hugo，纯静态网站，零成本，可持续更新

## 方案概述

| 方案 | 费用 | 难度 | 维护方式 |
|------|------|------|----------|
| **GitHub Pages + Hugo** | 免费 | 中等 | Git 推送 |
| Gitee Pages | 免费* | 中等 | 手动上传 |
| Vercel | 免费 | 简单 | Git 推送 |

*注：Gitee 免费版不支持私人仓库自动部署

---

## 第1步：环境准备

### 1.1 注册 GitHub 账号

1. 访问 [github.com](https://github.com)
2. 点击 **Sign up** 注册
3. 推荐用户名使用英文（如 `noeliu`）

### 1.2 安装 Git（Windows）

如果你已安装 MSYS2，直接运行：

```bash
pacman -S git
```

验证安装：
```bash
git --version
```

配置 Git：
```bash
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
```

### 1.3 安装 Hugo

**方式一：通过 MSYS2 安装（推荐）**
```bash
pacman -S hugo
```

**方式二：下载二进制**
- 访问 [github.com/gohugoio/hugo/releases](https://github.com/gohugoio/hugo/releases)
- 下载 `hugo_extended_xxx_windows-amd64.zip`
- 解压到 `C:\hugo` 并添加到系统 PATH

验证安装：
```bash
hugo version
```

## 第2步：创建网站

### 2.1 创建新站点

```bash
# 创建新网站（将 mysite 替换为你的网站目录名）
hugo new site mysite
cd mysite
```

### 2.2 添加主题

以 **LoveIt** 主题为例（界面美观，支持暗色模式）：

```bash
# 初始化 Git 仓库
git init

# 添加主题为 Git 子模块
git submodule add https://github.com/dillonzq/LoveIt.git themes/LoveIt
```

### 2.3 配置主题

编辑 `config.toml`：

```toml
baseURL = "https://example.com/"
languageCode = "zh-cn"
title = "我的个人网站"
theme = "LoveIt"

# 中文搜索功能
[languages.zh-cn.search]
  language = "zh-cn"

[params]
  version = "0.2.X"
  description = "我的个人网站"
  author = "Your Name"
```

### 2.4 创建示例内容

```bash
# 创建第一篇文章
hugo new posts/hello-world.md
```

编辑 `content/posts/hello-world.md`：

```markdown
---
title: "你好，世界"
date: 2026-05-11
draft: false
---

欢迎来到我的个人网站！这是一个使用 Hugo 和 GitHub Pages 构建的静态网站。
```

### 2.5 本地预览

```bash
# 启动本地服务器
hugo server

# 访问 http://localhost:1313 查看效果
```

按 `Ctrl + C` 停止预览服务器。

## 第3步：部署到 GitHub Pages

### 3.1 创建 GitHub 仓库

1. 登录 GitHub
2. 点击右上角 **+** → **New repository**
3. 仓库名填写：`noeliu.github.io`（替换为你的用户名）
4. 选择 **Public**
5. 点击 **Create repository**

### 3.2 推送到 GitHub

```bash
# 添加远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/noeliu/noeliu.github.io.git

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: 我的个人网站"

# 推送（首次需要设置上游分支）
git push -u origin main
```

### 3.3 启用 GitHub Pages

1. 进入仓库 **Settings** → **Pages**
2. **Source** 选择 `Deploy from a branch`
3. **Branch** 选择 `main` 和 `/ (root)`
4. 点击 **Save**

等待 2-3 分钟，网站即可访问：`https://noeliu.github.io`

## 第4步：自动化部署（推荐）

每次手动运行 `hugo` 再推送太麻烦，使用 GitHub Actions 自动构建部署。

### 4.1 创建工作流文件

在本地网站目录创建文件：

**创建目录：**
```bash
mkdir -p .github/workflows
```

**创建部署文件：**
```bash
cat > .github/workflows/deploy.yml << 'EOF'
name: Deploy Hugo site

on:
  push:
    branches: [main]

jobs:
  build-deploy:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          submodules: true
          fetch-depth: 0
      
      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
      
      - name: Build Hugo site
        run: hugo --minify
      
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
EOF
```

### 4.2 提交并推送

```bash
git add .
git commit -m "Add GitHub Actions workflow for auto deployment"
git push
```

### 4.3 查看部署状态

1. 进入 GitHub 仓库页面
2. 点击 **Actions** 标签页
3. 可以看到自动构建部署的状态

以后每次推送新内容，网站会自动更新！

## 第5步：个性化配置

### 5.1 修改网站信息

编辑 `config.toml`：

```toml
baseURL = "https://noeliu.github.io/"
languageCode = "zh-cn"
title = "NoeLiu's Blog"
theme = "LoveIt"

[params]
  description = "技术分享与个人成长"
  author = "NoeLiu"
```

### 5.2 添加关于页面

```bash
hugo new about.md
```

编辑 `content/about.md`：

```markdown
---
title: "关于我"
date: 2026-05-11
---

# 关于我

你好！我是 NoeLiu。

## 技术栈
- C/C++ 开发
- Windows 应用开发
- 正在学习 Web 开发

## 联系方式
- GitHub: https://github.com/NoeLiu
```

### 5.3 添加评论功能

使用 **Giscus**（基于 GitHub Discussions）：

1. 访问 [giscus.app](https://giscus.app)
2. 登录 GitHub 账号并授权
3. 选择要关联的仓库（启用 Discussions 功能）
4. 复制生成的代码
5. 添加到主题配置或 `layouts/partials/comments.html`

### 5.4 绑定自定义域名（可选）

如果你有域名（如 `example.com`）：

1. 在域名服务商添加 DNS 记录：
   - 添加 `CNAME` 记录指向 `noeliu.github.io`
2. 在 GitHub 仓库 **Settings** → **Pages** → **Custom domain** 输入你的域名
3. 勾选 **Enforce HTTPS**

## 常见问题

### Q1：GitHub 访问慢怎么办？

**方案：使用 Cloudflare Pages 加速**

1. 注册 [Cloudflare](https://cloudflare.com)
2. 创建 Pages 项目，连接你的 GitHub 仓库
3. 构建命令填 `hugo`，输出目录填 `public`
4. 获得加速后的域名（如 `xxx.pages.dev`）

### Q2：本地预览显示乱码？

确保 `config.toml` 设置了中文：
```toml
languageCode = "zh-cn"
```

### Q3：推送后网站没更新？

1. 检查 GitHub Actions 是否运行成功
2. 确认分支名是 `main`（如果不是，修改 workflow 文件中的分支名）

### Q4：如何修改主题样式？

在 `config.toml` 的 `[params]` 中添加自定义配置，或创建 `assets/css/custom.css`。
